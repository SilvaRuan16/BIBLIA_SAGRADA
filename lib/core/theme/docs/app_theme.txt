Documentação do Sistema de Temas (AppTheme)
1. Conceito Estrutural

O sistema foi projetado para separar a estética global (fundo, textos e barras) da lógica de ação (botões positivos, negativos e padrão). Isso permite que o aplicativo mantenha uma identidade visual rigorosa enquanto oferece acessibilidade e conforto visual através dos modos claro e escuro.
2. Camada de Personalização: AppUiColors

Como o Flutter possui um conjunto limitado de cores semânticas, utilizamos o ThemeExtension. Esta classe permite injetar propriedades de cores personalizadas diretamente no ThemeData.
Atributos de Ação

    actionStandard: Azul vibrante utilizado para navegação e fluxo principal.

    actionPositive: Verde esmeralda para confirmações (ex: salvar, marcar como lido).

    actionNegative: Vermelho coral para ações destrutivas (ex: apagar marcador, excluir download).

Métodos Técnicos

    copyWith: Mantém a imutabilidade do tema, permitindo modificações parciais.

    lerp: Implementa a Interpolação Linear. É o que garante que as cores de ação "deslizem" suavemente entre um tom e outro durante a troca de tema, em vez de piscarem bruscamente.

3. Configurações Globais (Design Tokens)

A classe AppTheme define como cada componente se comporta nativamente, reduzindo a necessidade de estilização manual nos Widgets.
3.1 Tema Claro (Light Mode)

    Body & AppBar: Definidos como 0xFFF8FAFC (Slate 50), um branco acinzentado moderno que reduz a fadiga ocular.

    Tipografia & Ícones: Utilizam o tom 0xFF0F172A (Slate 900) para contraste máximo.

    Botões: Fundo azul profundo com texto/ícones em branco gelo.

3.2 Tema Escuro (Dark Mode)

    Body & AppBar: Definidos como 0xFF0F172A. O uso de um azul marinho profundo em vez do preto puro cria uma estética "Premium" e melhora a percepção de profundidade.

    Tipografia & Ícones: Utilizam 0xFFF1F5F9 (Slate 100).

    Botões: Utilizam uma cor pastel (0xFFD1E4FF) com texto/ícones em azul escuro, garantindo legibilidade em ambientes de baixa luz.

4. Automatização de Componentes

O código configura sub-temas específicos para garantir que componentes complexos já nasçam estilizados:

    AppBarTheme: Centraliza títulos e remove sombras (elevation 0) por padrão.

    IconTheme: Sincroniza a cor de todos os ícones do app com a cor do texto principal.

    ElevatedButtonTheme: Define que todo botão terá automaticamente a cor correta de fundo e a cor de contraste para o texto interno, eliminando erros de design.

5. Facilidade de Consumo: ThemeGetter

Foi criada uma extensão no BuildContext para tornar o código da View (no padrão MVC) muito mais limpo.
Tipo de Acesso	Sintaxe
Cores Globais	context.colorScheme.primary
Fundo do App	context.colorScheme.surface
Cores de Ação	context.uiColors.actionPositive

Exemplo Prático na View:
Dart

// Um ícone de leitura que se ajusta sozinho ao tema
Icon(Icons.menu_book, color: context.colorScheme.onSurface)

// Um botão de excluir que usa a cor de ação negativa
ElevatedButton(
  style: ElevatedButton.styleFrom(backgroundColor: context.uiColors.actionNegative),
  onPressed: () {},
  child: const Text("Remover"),
)

6. Conclusão

Esta arquitetura garante que a manutenção visual seja feita em um único ponto. Se a marca decidir mudar o tom de azul, a alteração em AppTheme propagará a mudança por todo o aplicativo, incluindo bordas de botões, ícones e estados de foco.
